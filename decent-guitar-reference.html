🎸 Decent Guitar

Project Reference — last updated June 2026

Overview

Decent Guitar is a personal web app for supporting acoustic guitar learning. It provides a song library with chord-annotated scrolling lyrics (similar to Ultimate Guitar's chord view), a chord fingering library, strumming pattern reference, and a daily practice module (in progress).

Database Schema

parts

Reference table. Shared part names reused across all songs. Seeded with standard values.

ColumnTypeNotes
PK idserialAuto-incrementing integer
nametextUnique. e.g. Intro, Verse, Chorus, Bridge, Pre-Chorus, Solo, Outro

strumming_patterns

ColumnTypeNotes
PK idserial
notationtexte.g. D DU UDU
notestextOptional description

chords

ColumnTypeNotes
PK idserial
nametextUnique. e.g. Em, G, C
fingeringjsonbPer-string: plucked (bool), finger (int|null), fret (int|null). Keys: e, a, d, g, b, high_e
notestextOptional. e.g. barre chord notes

Example fingering JSON (Em):

{"e":{"plucked":true,"finger":null,"fret":null},"a":{"plucked":true,"finger":1,"fret":2},...}

songs

ColumnTypeNotes
PK iduuidgen_random_uuid()
titletext
artisttextNullable
capo_fretintNullable. 1–12
source_urltextNullable. e.g. Ultimate Guitar link
notestextNullable
FK default_strumming_pattern_idintNullable. Points to strumming_patterns. Used when same pattern throughout.

song_parts_order

Intermediate table. Assigns a song-specific ordering to part instances (e.g. Verse appears at position 2 and 4 in the same song). Unique constraint on (song_id, part_order).

ColumnTypeNotes
PK iduuid
FK song_iduuid→ songs, cascade delete
FK part_idint→ parts
part_orderint1-based sequence within a song

song_part_strumming

Overrides the song-level default strumming pattern for a specific part instance.

ColumnTypeNotes
FK song_parts_order_iduuidComposite PK with strumming_pattern_id
FK strumming_pattern_idint→ strumming_patterns

song_lyrics

One row per lyric line. Chord changes are embedded inline using [ChordName] notation (e.g. [Em]Never gonna [G]give you up). Chord list for display is derived at query time by parsing these markers — no separate song_part_chords table needed.

ColumnTypeNotes
PK iduuid
FK song_parts_order_iduuid→ song_parts_order, cascade delete
line_orderint1-based sequence within a part. Unique with song_parts_order_id.
lyric_texttextRaw lyric line with inline [Chord] markers

Deployment Workflow

  1. Download updated file(s) from Claude
  2. Go to github.com/danielgomez909/decent-guitar
  3. Open the file → click pencil (edit) icon, paste new content
  4. Commit changes — Vercel auto-deploys in ~30 seconds
  5. Verify at the Vercel URL
